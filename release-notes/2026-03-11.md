# Release Notes - 2026-03-11

## Feature Name
- Docker runtime stability + dashboard resilience for store sync/import flows

## What’s New
- Fixed Streamlit entrypoint execution so dashboard UI reliably renders in Docker (`src/run_dashboard.py` now executes `iris_dashboard.main()`).
- Added defensive handling in store drill-down to prevent `KeyError: 'camera_id'` and related gallery rendering failures when partial data is present.
- Hardened Google Drive sync behavior for non-JSON API responses with clearer error messages and improved shared-drive file listing options.
- Improved Drive fallback behavior by allowing partial folder download progress when using `gdown`.

## Impact
- Improves user experience by preventing blank/failed dashboard render states after deployment.
- Reduces operational errors during store onboarding, TSV import, and Drive-linked sync refreshes.
- Supports faster pilot rollout by providing actionable sync diagnostics instead of ambiguous exceptions.

## Metrics / Monitoring
- KPI being tracked: Drive sync success rate per store; dashboard runtime error count.
- Expected improvement: Lower sync failure confusion and fewer uncaught UI exceptions.
- Dashboard / Tool used: Streamlit app logs (`docker compose ... logs -f iris`) and in-app sync status messages.
- Monitoring owner: IRIS engineering / operations owner.

## Availability
- Web
- Release version (if applicable): main @ 2026-03-11 patch set

## Risks / Known Issues
- Large/shared Drive folders may still require a valid `GOOGLE_API_KEY` and correct sharing permissions for consistent full sync.
- `use_container_width` warnings from Streamlit remain informational and do not block runtime.

## Rollback Plan
- Revert latest patch commits on `main` and redeploy container:
  - `git revert <commit_sha>`
  - `docker compose -f deploy/docker-compose.yml up --build -d`
